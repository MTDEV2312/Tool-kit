# Team Conventions & Development Workflow

This document records the development rules and conventions agreed upon for this repository.

## 🚀 Commit & Release Workflow Rule
- **Rule**: Once a feature is fully developed, formatted, and verified by passing all unit and integration tests, you **MUST** immediately stage all changes, create a descriptive conventional commit, and push (`git push`) the changes to the remote repository before beginning the next task.
- **Goal**: Maintain a clean, readable, versioned, and synchronized branch state where every completed feature is instantly integrated and shared.
